Header files:
it is a file that contains the declaration(ie just like .pyi files in python) of functions, classes etc..
its required by the compiler because of the static nature of c++
the actual code of the functions in the header file is located in a seprate .cpp file

All the C++ standard library functions,objects etc are stored under the std namespace
whereas all the C standard library functions, objects etc are stored under both std and global namespace
(any module with a prefix of "c" is a part of C standard library)
thats why when you include modules like cmath you dont need to use the scope resolution operator to
access its functions

Symbol Table:
used to store various information about the program, used by compliers during compilation process
(but the final executable binary doesnt have any symbol table) to map variable names(including their data types)
to a corresponding memory address where their value is located

example:
int x = 1025; => Symbol table entry for this line = int "x": 101

    ↑
+--------+        |The diagram in the side is used to represent computer memory(RAM)
|00000001|101     |101 to 104 are the memory allocated to x
+--------+        |each cell represents a byte and corresponding to it is its memory address
|00000100|102     |all these 4 bytes combined are used to represent the value 1025
+--------+        |4 bytes were used because int datatype is of 4 byets
|00000000|103
+--------+
|00000000|104
+--------+
|        |105
+--------+
    ↓
